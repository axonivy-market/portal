複合フィルター 
*****************************

概要
------------
ポータルには、見やすいように、直感的で分かりやすい、強力なフィルタリングシステムが備わっており、特定の基準に従って表示をカスタマイズできます。
この機能は、ダッシュボードのケースウィジェットとタスクウィジェットで利用可能です。

使用方法
----------

- ダッシュボードのタスク／ケースウィジェットにある :guilabel:`フィルター` ボタンをクリックするか、ダッシュボードの編集のウィジェットの編集設定パネルにある :guilabel:`フィルター` ボタンをクリックして、フィルターダイアログにアクセスできます。 
  詳細については、:ref:`new-dashboard-ja` を参照してください。 

- フィルターを追加するには、 ``フィルターを追加`` を選択し、列と演算子を選択し、必要に応じて値を入力します。  
    
    |filter-operator-dropdown|

- ポータルではフィルターを組み合わせることができます。 **AND** を使用して、組み合わせるフィルターを互いに連結します。

    |case-task-widget-filter-combine|

- |remove-filter-icon| ボタンをクリックして、フィルターを削除します。
- ``適用`` ボタンをクリックして、フィルターを適用します。

.. tip::
    例えば、ケースウィジェットで、次のフィルターを適用します：

      |complex-filter-example|

    このフィルター設定では、ケース名に「Mike」を含む、今日作成された、状態が未完了のすべてのケースが検索されます。

ダッシュボードの設定で、タスク／ケースウィジェットにフィルターが事前に設定されている場合、それらのフィルターが読み取り専用フィルターとして表示されます。

フィルターの演算子の詳細については、:ref:`filter-operators-ja` を参照してください。

フィルターセットの保存 
----------------------------------------

ダッシュボードのタスク／ケースウィジェットでは、以下の手順に従って、追加したフィルターをフィルターセットに保存できます。

- ``フィルターを保存`` ボタンをクリックし、フィルターセットの名前を入力し、 ``保存`` をクリックします。

- 保存されたフィルターセットは、フィルターダイアログの左側に表示されます。

- フィルターセットを選択して適用し、選択解除してフィルターセットを削除します。

- フィルターセットの名前が太字の場合、そのフィルターセットが現在適用されています。

    |widget-save-filter|

フィルターセットの管理
-----------------------------------

ウィジェットフィルター管理ダイアログで保存されたフィルターセットを管理できます。

- ``利用可能なフィルター`` ダイアログで ``フィルターの管理`` を選択します。

- フィルターセットを検索するには、各列の一番上にある入力フィールドに検索値を入力します。

- フィルターを削除するには、フィルターセットの隣にあるチェックボックスを選択し、:guilabel:`選択から削除` ボタンをクリックします。

    |widget-filter-management|

.. _filter-operators-ja:

フィルターの演算子
----------------------------

.. list-table::

 * - **フィルター**
   - **利用ガイドライン**
 * - 指定の値以内
   - | カテゴリー、状態、作成者、アプリケーションに有効。1 つの値または複数の値を指定します。
     | 結果には、列の値がフィルターの値に一致するレコードのみ含まれます。
 * - 指定の値以外
   - | カテゴリー、作成者に有効。1 つの値または複数の値を指定します。  
     | 結果には、列の値がフィルターの値に一致しないレコードのみ含まれます。
 * - 指定の値を含む
   - | カテゴリー、ID、テキストを含む列に有効。1 つの値または複数の値を指定します。  
     | 結果には、列のデータにフィルターの値が含まれるレコードのみ含まれます。
 * - 指定の値を含まない
   - | カテゴリー、ID、テキストを含む列に有効。1 つの値または複数の値を指定します。  
     | 結果には、列のデータにフィルターの値が含まれないレコードのみ含まれます。
 * - 指定の値の間にある
   - | 数値、日付を含む列に有効。2 つの値を指定します。  
     | 結果には、列のデータがフィルターの 2 つの値の間にあるレコードのみ含まれます。
 * - 指定の値の間にない
   - | 数値、日付を含む列に有効。2 つの値を指定します。  
     | 結果には、列のデータがフィルターの 2 つの値の間にないレコードのみ含まれます。
 * - 指定の値と等しい
   - | 数値を含む列に有効。1 つの値を指定します。  
     | 結果には、列の値がフィルターの値に一致するレコードのみ含まれます。
 * - 指定の値と等しくない
   - | 数値を含む列に有効。1 つの値を指定します。  
     | 結果には、列の値がフィルターの値に一致しないレコードのみ含まれます。
 * - 指定の値より小さい
   - | 数値を含む列に有効。1 つの値を指定します。  
     | 結果には、列の値がフィルターの値より小さいレコードのみ含まれます。
 * - 指定の値より小さいか等しい
   - | 数値を含む列に有効。1 つの値を指定します。  
     | 結果には、列の値がフィルターの値より小さいか等しいレコードのみ含まれます。
 * - 指定の値より大きい
   - | 数値を含む列に有効。1 つの値を指定します。  
     | 結果には、列の値がフィルターの値より大きいレコードのみ含まれます。
 * - 指定の値より大きいか等しい
   - | 数値を含む列に有効。1 つの値を指定します。  
     | 結果には、列の値がフィルターの値より大きいか等しいレコードのみ含まれます。
 * - 空欄である
   - | テキスト、数値、日付を含む列に有効。値は指定しないでください。
     | 結果には、列にデータが含まれないレコードのみ含まれます。
 * - 空欄ではない
   - | テキスト、数値、日付を含む列に有効。値は指定しないでください。
     | 結果には、列にデータが含まれるレコードのみ含まれます。
 * - 指定の値である
   - | テキスト、日付を含む列に有効。1 つの値または複数の値を指定します。
     | 結果には、列の値がフィルターの値に一致するレコードのみ含まれます。
 * - 指定の値ではない
   - | テキスト、日付を含む列に有効。1 つの値または複数の値を指定します。
     | 結果には、列の値がフィルターの値に一致しないレコードのみ含まれます。
 * - 今日
   - | 日付を含む列に有効。値は指定しないでください。
     | 結果には、列の日付が現在の日付に一致するレコードのみ含まれます。
 * - 昨日
   - | 日付を含む列に有効。値は指定しないでください。
     | 結果には、列の日付が昨日の日付に一致するレコードのみ含まれます。
 * - 指定の値の前にある
   - | 日付を含む列に有効。1 つの値を指定します。
     | 結果には、列の日付がフィルターの日付の前にあるレコードのみ含まれます。
 * - 指定の値の後にある
   - | 日付を含む列に有効。1 つの値を指定します。
     | 結果には、列の日付がフィルターの日付の後にあるレコードのみ含まれます。
 * - 現在
   - | 日付を含む列に有効。年、月または週を選択します。
     | 結果には、列の日付がフィルターで指定した範囲内にあるレコードのみ含まれます。
     | 具体的には、現在の年／月／週の最初の日から現在の年／月／週の最後の日の終わりまで
     | となります。
     | 
     | *例*：現在の月に作成されたすべてのケースを検索します。
     |  :bdg-dark-line:`作成日` :bdg-dark-line:`現在` :bdg-dark-line:`月`
     |  
     | *説明*：今日が ``2024/2/27 11:41:28`` であると仮定した場合、クエリは、作成日が 
     | ``2024/2/1 00:00:00`` から ``2024/2/29 23:59:59`` までの期間のケースを検索します。
 * - 今日までの期間
   - | 日付を含む列に有効。1 つの値を指定し、年、月、週または日を選択します。
     | 結果には、列の日付がフィルターで指定した範囲内にあるレコードのみ含まれます。
     | 具体的には、過去 <input_value> 年／月／週の最初の日の始まりから 
     | 現在までとなります。
     |
     | *例*：過去 2 年間に作成されたすべてのケースを検索します。
     |  :bdg-dark-line:`作成日` :bdg-dark-line:`今日までの期間` :bdg-dark-line:`2` :bdg-dark-line:`年`
     |  
     | *説明*：今日が ``2024/2/27 11:41:28`` であると仮定した場合、クエリは、作成日が 
     | ``2022/2/27 00:00:00`` から ``2024/2/27 11:41:28`` までの期間のケースを検索します。
 * - 次の期間
   - | 日付を含む列に有効。1 つの値を指定し、年、月、週または日を選択します。
     | 結果には、列の日付がフィルターで指定した範囲内にあるレコードのみ含まれます。
     | 具体的には、現在から次の <input_value> 年／月／週間の日の終わりまで 
     | となります。
     |
     |  *例* ：請求書の支払期日が次の 3 週間以内のすべてのケースを検索します。
     | :bdg-dark-line:`請求書の支払期日` :bdg-dark-line:`次の期間` :bdg-dark-line:`3` :bdg-dark-line:`週`
     |  
     |  *説明* ：今日が ``2024/2/27 11:41:28`` であると仮定した場合、クエリは、請求書の支払期日が 
     | ``2024/2/27 11:41:28`` から ``2024/3/19 23:59:59`` までの期間のケースを検索します。
 * - 指定の値で始まる
   - | テキストを含む列に有効。1 つの値または複数の値を指定します。
     | 結果には、列のデータがフィルターの値で始まるレコードのみ含まれます。
 * - 指定の値で始まらない
   - | テキストを含む列に有効。1 つの値または複数の値を指定します。
     | 結果には、列のデータがフィルターの値で始まらないレコードのみ含まれます。
 * - 指定の値で終わる
   - | テキストを含む列に有効。1 つの値または複数の値を指定します。
     | 結果には、列のデータがフィルターの値で終わるレコードのみ含まれます。
 * - 指定の値で終わらない
   - | テキストを含む列に有効。1 つの値または複数の値を指定します。
     | 結果には、列のデータがフィルターの値で終わらないレコードのみ含まれます。
 * - カテゴリーなし
   - | カテゴリーに有効。値は指定しないでください。
     | 結果には、列のデータがカテゴリーなしのレコードのみ含まれます。
 * - 現在のユーザー
   - | 作成者に有効。値は指定しないでください。
     | 結果には、列のデータが現在のユーザーによって作成されたレコードのみ含まれます。
